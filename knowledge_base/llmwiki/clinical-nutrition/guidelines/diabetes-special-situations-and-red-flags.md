# 糖尿病营养建议的特殊情况与红旗信号

## 必须先问清的情况

当用户提到糖尿病饮食、控糖、减重或运动时，Agent 应优先识别以下信息：

- 是否正在使用胰岛素、磺脲类或其他可能导致低血糖的药物。
- 是否有糖尿病肾病、肾功能不全、蛋白尿或医生限制蛋白质。
- 是否近期反复低血糖、夜间低血糖或血糖大幅波动。
- 是否合并高血压、血脂异常、痛风、脂肪肝、冠心病。
- 是否妊娠、备孕、哺乳或儿童青少年。

## 红旗信号

出现以下信息时，不应继续普通营养建议，应提示尽快联系医生或急诊：

- 明显低血糖表现：心慌、出汗、手抖、意识模糊、昏厥。
- 严重高血糖伴口渴、多尿、乏力、恶心呕吐、呼吸异常或酮症风险。
- 足部破溃、感染、发热或伤口不愈合。
- 用户自行大幅调整胰岛素或降糖药剂量。
- 肾功能不全用户准备高蛋白饮食、低钠盐或高钾盐。

## 药物与饮食交互

糖尿病用户常同时使用降压药、降脂药或抗凝药。Agent 在饮食建议里应注意强规则引擎已覆盖的高风险组合：

- 西柚或葡萄柚 + 部分钙通道阻滞剂或他汀类药物。
- 酒精 + 头孢类抗生素或甲硝唑。
- 高维生素 K 食物突然大幅变化 + 华法林。
- 低钠盐或高钾盐 + 肾功能不全 + 普利类或沙坦类药物。

## 糖尿病肾病提示

如果用户有糖尿病肾病或肾功能不全，不要泛化推荐“高蛋白增肌”“多吃豆制品”“低钠盐替代普通盐”。此类用户的蛋白质、钾、磷、钠需要根据肾功能分期和医生方案个体化。

## 运动与低血糖

如果用户询问餐后运动，建议结合药物和血糖状态。使用胰岛素或易低血糖药物者，运动前后应更注意血糖监测和低血糖处理准备。Agent 不应在不知道药物情况下建议空腹高强度运动。

## Agent 回复策略

遇到红旗信号时，回复结构应为：

1. 先确认风险，而不是继续算营养。
2. 给出立即行动建议，例如测血糖、补充快速糖源或尽快就医，具体取决于用户描述。
3. 明确说明线上营养建议不能替代急性处理。
4. 后续再回到饮食复盘。
